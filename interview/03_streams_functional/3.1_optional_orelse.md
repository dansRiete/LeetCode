# 11. What is the difference between Optional.orElse() and Optional.orElseGet()?

## Your Answer
- orElse() returns the value given as argument ✓
- orElseGet() accepts a function (Supplier) ✓
- Would use orElseGet() for expensive computations to avoid eager evaluation ✓

## What Was Missing / Wrong

- Missing: the precise terminology — **eager** (orElse) vs **lazy** (orElseGet)

## Complete Answer

### orElse() — eager, always evaluated:
```java
// callDatabase() is ALWAYS called even if optional has a value
String result = optional.orElse(callDatabase());
```

### orElseGet() — lazy, only called when Optional is empty:
```java
// callDatabase() only called when optional is empty
String result = optional.orElseGet(() -> callDatabase());
```

### Rule of thumb:

- `orElse(value)` — use for cheap, already-computed values: `null`, `""`, `0`, constants
- `orElseGet(() -> value)` — use for expensive: DB calls, API calls, object construction

### Other Optional methods worth knowing:
```java
optional.orElseThrow()                              // throw NoSuchElementException if empty
optional.orElseThrow(() -> new NotFoundException()) // throw custom exception if empty
optional.map(Flight::price)                         // transform if present
optional.filter(f -> f.price() > 100)              // filter, returns empty if false
optional.ifPresent(f -> process(f))                 // execute if present, void
optional.ifPresentOrElse(
    f -> process(f),
    () -> handleEmpty()                             // both branches
);
```
