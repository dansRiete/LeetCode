## Complete Answer

### What is a functional interface:
An interface with **exactly one abstract method** — annotated with `@FunctionalInterface`.
This makes it usable as a lambda expression.

### Top 5 from java.util.function:

- `Function<T,R>` — signature: T → R;
  example: `s -> s.toUpperCase()`;
  use case: transform a value
- `Consumer<T>` — signature: T → void;
  example: `s -> System.out.println(s)`;
  use case: side effects, logging
- `Supplier<T>` — signature: () → T;
  example: `() -> new Flight()`;
  use case: lazy value production
- `Predicate<T>` — signature: T → boolean;
  example: `s -> s.length() > 5`;
  use case: filtering
- `BiFunction<T,U,R>` — signature: T,U → R;
  example: `(a,b) -> a + b`;
  use case: combine two values

### Also worth knowing:
```java
UnaryOperator<T>    // Function<T,T> — input and output same type
BinaryOperator<T>   // BiFunction<T,T,T> — two inputs, same type as output
BiConsumer<T,U>     // takes two args, returns void
BiPredicate<T,U>    // takes two args, returns boolean
```

### Usage in streams:
```java
List<Flight> flights = ...;

flights.stream()
    .filter(f -> f.price() > 100)          // Predicate
    .map(Flight::destination)               // Function
    .forEach(System.out::println);          // Consumer

flights.stream()
    .collect(Collectors.groupingBy(
        Flight::destination,                // Function (classifier)
        Collectors.mapping(
            Flight::name,                   // Function (mapper)
            Collectors.toList()
        )
    ));
```

### Memory trick:
- **S**upplier — **S**upplies (produces) a value, takes nothing
- **C**onsumer — **C**onsumes a value, returns nothing
- **F**unction — transforms (maps) T to R
- **P**redicate — tests a condition, returns boolean
