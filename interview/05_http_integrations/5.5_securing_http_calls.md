## What Was Missing / Wrong
- All three need to be known for a senior integration role

## Complete Answer

### 1. API Keys — simplest:
```java
conn.setRequestProperty("X-Api-Key", "your-api-key");
// or as query param: GET /bookings?apiKey=your-key
```
- Static key, easy to implement
- Risk: if leaked anyone can use it — rotate regularly
- Store in **GCP Secret Manager**, never hardcode

### 2. OAuth2 Client Credentials — most common for integrations:
```
Step 1: POST /oauth/token
        Body: grant_type=client_credentials&client_id=xxx&client_secret=yyy

Step 2: Response: { "access_token": "abc123", "expires_in": 3600 }

Step 3: Attach to every request:
        Authorization: Bearer abc123

Step 4: When token nears expiry → request new one proactively
```

```java
public class TokenCache {
    private String token;
    private Instant expiry;

    public String getToken() {
        if (token == null || Instant.now().isAfter(expiry.minusSeconds(60))) {
            refreshToken(); // refresh 60s before expiry
        }
        return token;
    }
}
```

### 3. mTLS (Mutual TLS) — most secure:
- Both client AND server present certificates
- Server verifies your identity, you verify server's identity
- Used in high-security integrations (banking, healthcare)
```java
SSLContext sslContext = SSLContext.getInstance("TLS");
sslContext.init(keyManagers, trustManagers, null);
HttpsURLConnection conn = (HttpsURLConnection) url.openConnection();
conn.setSSLSocketFactory(sslContext.getSocketFactory());
```

### Which to expect at this role:

- Modern partner APIs → OAuth2 Client Credentials
- Simpler/legacy partners → API Keys
- High security partners → mTLS
- Public APIs → API Keys or OAuth2

### Senior points:
- Never hardcode credentials — use **GCP Secret Manager**
- OAuth2 tokens must be **cached** — requesting new token per call wastes resources
  and may get rate limited by auth server
- Refresh token **proactively** (60s before expiry), not reactively (after 401)
- API key rotation must not require downtime — support two active keys during
  rotation window
