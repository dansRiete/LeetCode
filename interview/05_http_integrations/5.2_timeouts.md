# 14. How do you handle timeouts — connection timeout vs read timeout?

## Your Answer
- Connection timeout = waiting for server to respond for too long ✗
- Read timeout = server hangs up / processes slowly ✗

## What Was Missing / Wrong
- **Definitions were swapped** — common mix-up
- Connection timeout = establishing the TCP connection
- Read timeout = waiting for response AFTER connection is established
- Missing: always set both, never leave at 0 (default = wait forever)

## Complete Answer

### Correct definitions:

**Connection timeout** — time to **establish the connection** (TCP handshake):
- Server doesn't respond to initial connection attempt
- Happens when: server is down, wrong IP, firewall blocking
- Analogy: phone keeps ringing, nobody picks up

**Read timeout** — time to **receive response after connection is established**:
- Server accepted connection but takes too long to send data
- Happens when: server overloaded, slow query, large payload
- Analogy: someone picks up but goes silent

### Plain Java:
```java
HttpURLConnection conn = (HttpURLConnection) url.openConnection();
conn.setConnectTimeout(3_000);   // 3s to establish TCP connection
conn.setReadTimeout(10_000);     // 10s to receive response after connected
```

### Sizing guidelines:

- Connection timeout: recommended 2-5s;
  reasoning: if unreachable in 3s, won't be in 30s
- Read timeout: depends on operation;
  reasoning: Search: 5s, Report: 60s

### Never leave at 0:
```java
// DEFAULT — waits forever
conn.setConnectTimeout(0); // infinite wait
conn.setReadTimeout(0);    // infinite wait

// In production this exhausts your thread pool:
// 50 threads all waiting indefinitely → no threads left for new requests
```

### Always set both — in integration context:
```java
public HttpURLConnection openConnection(URL url) throws IOException {
    HttpURLConnection conn = (HttpURLConnection) url.openConnection();
    conn.setConnectTimeout(3_000);
    conn.setReadTimeout(10_000);
    return conn;
}
```

### Memory trick:
- **Connect** = knocking on the door
- **Read** = waiting for the person to answer after they opened the door
