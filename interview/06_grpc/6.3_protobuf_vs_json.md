## Complete Answer

### Serialization size & speed:

- Format: JSON is text; Protobuf is binary
- Human readable: JSON yes; Protobuf no
- Field names in payload: JSON yes (verbose); Protobuf no (field numbers only)
- Typical size difference: JSON is baseline; Protobuf is 3-10x smaller
- Parse speed: JSON is slower; Protobuf is much faster

### How Protobuf encodes data:
```protobuf
message Flight {
    int32 passenger_id = 1;   // field number 1
    string destination = 2;   // field number 2
    int32 price = 3;          // field number 3
}
```
Wire format only contains field numbers + values — no field names.
`{ "passenger_id": 123, "destination": "LAX", "price": 350 }` in JSON
vs just `[1:123, 2:"LAX", 3:350]` in binary.

### Schema evolution — safe changes:
```protobuf
// V2 — add new field safely
message Flight {
    int32 passenger_id = 1;
    string destination = 2;
    int32 price = 3;
    string currency = 4;   // new — old clients ignore unknown field numbers
}
```

- Add new field → safe: old clients ignore unknown numbers
- Remove a field → safe if reserved: must reserve number to prevent reuse
- Rename a field → safe: wire format uses numbers, not names
- Change field type → NOT safe: corrupts deserialization
- Reuse a field number → NOT safe: old clients misinterpret data

### The reserved keyword — critical:
```protobuf
message Flight {
    int32 passenger_id = 1;
    string destination = 2;
    // price was removed — MUST reserve both number and name
    reserved 3;
    reserved "price";
    // now field number 3 and name "price" can never be reused
}
```

### vs JSON schema evolution:
- JSON has no built-in schema versioning — rely on `FAIL_ON_UNKNOWN_PROPERTIES=false`
- No formal contract — schema drift is harder to detect
- Renaming a JSON field IS a breaking change (field name is the identifier)

### Key point:
Protobuf schema in `.proto` files is a **formal contract** — both sides must agree
upfront. This is a strength (catches breaking changes early) and a constraint
(requires coordination).
