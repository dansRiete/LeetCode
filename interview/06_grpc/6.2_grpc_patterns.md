## Complete Answer

### 1. Unary — simplest, like REST:
```protobuf
rpc GetFlight(FlightRequest) returns (FlightResponse);
```
```
Client → single request → Server
Client ← single response ← Server
```
Use case: get a booking, search for flights

### 2. Server Streaming — server pushes multiple responses:
```protobuf
rpc StreamFlightUpdates(FlightRequest) returns (stream FlightResponse);
```
```
Client → single request → Server
Client ← response 1 ←
Client ← response 2 ←
Client ← response 3 ←
```
Use case: partner pushes live flight status updates, price changes

### 3. Client Streaming — client pushes multiple requests:
```protobuf
rpc UploadBookings(stream BookingRequest) returns (UploadResult);
```
```
Client → request 1 →
Client → request 2 → Server
Client → request 3 →
Client ← single response ←
```
Use case: bulk upload of booking records to partner

### 4. Bidirectional Streaming — both stream simultaneously:
```protobuf
rpc SyncBookings(stream BookingRequest) returns (stream BookingResponse);
```
```
Client → request 1 →
Client ← response 1 ←
Client → request 2 → Server
Client ← response 2 ←
```
Use case: real-time sync between two systems

### Memory trick:
- **Unary** = one request, one response (like REST)
- **Server streaming** = one request, many responses (server pushes)
- **Client streaming** = many requests, one response (client pushes)
- **Bidirectional** = many requests, many responses (both push)

### Most relevant for integration work:
- **Unary** — most common day-to-day
- **Server streaming** — partner pushing live updates (pricing, availability)
