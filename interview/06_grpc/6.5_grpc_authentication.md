# 25. How do you add authentication to a gRPC call?

## Your Answer
- Not sure

## What Was Missing / Wrong
- Authentication in gRPC is done via **interceptors** and **Metadata**
  (equivalent of HTTP headers)

## Complete Answer

### Key concepts:
- **Metadata** = gRPC equivalent of HTTP headers
- **Interceptors** = gRPC equivalent of HTTP middleware/filters
- Apply interceptor once → works for every call automatically

### Client side — attach token to every call:
```java
ClientInterceptor authInterceptor = new ClientInterceptor() {
    @Override
    public <Q, R> ClientCall<Q, R> interceptCall(
            MethodDescriptor<Q, R> method, CallOptions options, Channel next) {
        return new ForwardingClientCall.SimpleForwardingClientCall<>(
                next.newCall(method, options)) {
            @Override
            public void start(Listener<R> responseListener, Metadata headers) {
                headers.put(
                    Metadata.Key.of("authorization", Metadata.ASCII_STRING_MARSHALLER),
                    "Bearer " + tokenCache.getToken()
                );
                super.start(responseListener, headers);
            }
        };
    }
};

// attach to channel — applies to all calls
ManagedChannel channel = ManagedChannelBuilder
    .forAddress("partner-api.com", 443)
    .intercept(authInterceptor)
    .build();
```

### Server side — validate token:
```java
ServerInterceptor authServerInterceptor = new ServerInterceptor() {
    @Override
    public <Q, R> ServerCall.Listener<Q> interceptCall(
            ServerCall<Q, R> call, Metadata headers, ServerCallHandler<Q, R> next) {
        String token = headers.get(
            Metadata.Key.of("authorization", Metadata.ASCII_STRING_MARSHALLER)
        );
        if (!isValid(token)) {
            call.close(
                Status.UNAUTHENTICATED.withDescription("Invalid token"), headers
            );
            return new ServerCall.Listener<>() {};
        }
        return next.startCall(call, headers);
    }
};
```

### Analogies to HTTP:

- HTTP Headers → gRPC Metadata
- HTTP Filter / Middleware → gRPC Interceptor
- HTTP `Authorization: Bearer xxx` → gRPC `Metadata.Key.of("authorization")`

### Same OAuth2 flow applies:
- Get token from auth server
- Cache it, refresh before expiry
- Attach via interceptor to every call
- Same `TokenCache` class works for both HTTP and gRPC

### Senior points:
- Use interceptors — don't manually add auth to each stub call
- Token caching is critical — same as HTTP, don't request new token per call
- For mTLS: configure `SslContext` on the `ManagedChannelBuilder` instead of
  header-based auth
