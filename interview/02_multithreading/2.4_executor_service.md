# 9. What is the difference between ExecutorService, ScheduledExecutorService, and ForkJoinPool?

## What Was Missing / Wrong
- All three need to be known for a senior integration engineer role

## Complete Answer

### ExecutorService — base thread pool:
```java
ExecutorService pool = Executors.newFixedThreadPool(10);
Future<Result> future = pool.submit(() -> callPartnerApi());
pool.shutdown();
```
- Manages a pool of threads
- Use for: parallel HTTP calls, processing records concurrently
- Key implementations: `FixedThreadPool`, `CachedThreadPool`, `SingleThreadExecutor`

### ScheduledExecutorService — adds time-based scheduling:
```java
ScheduledExecutorService scheduler = Executors.newScheduledThreadPool(5);

// run once after delay
scheduler.schedule(() -> retryFailedCall(), 5, TimeUnit.SECONDS);

// run repeatedly at fixed rate
// (starts next run regardless of previous completion)
scheduler.scheduleAtFixedRate(() -> pollPartnerApi(), 0, 30, TimeUnit.SECONDS);

// run repeatedly with fixed delay
// (waits for previous to complete before counting delay)
scheduler.scheduleWithFixedDelay(() -> pollPartnerApi(), 0, 30, TimeUnit.SECONDS);
```
- Use for: polling partner APIs, delayed retries, OAuth token refresh, cache refresh

### ForkJoinPool — divide and conquer:
```java
ForkJoinPool pool = new ForkJoinPool(4);
pool.submit(() ->
    largeList.parallelStream().map(this::process).toList()
);
```
- Uses **work stealing** — idle threads steal tasks from busy threads' queues
- Powers `parallelStream()` under the hood
- Use for: CPU-intensive recursive tasks, parallel data processing

### Key warning about ForkJoinPool:
- `parallelStream()` uses the **common ForkJoinPool** shared by the entire JVM
- Blocking operations (HTTP calls) in `parallelStream()` can starve other tasks
- For I/O-bound work (integrations) use `ExecutorService`, not `parallelStream()`

### Integration context usage:

- Fan out to multiple partners → `ExecutorService`
- Poll partner API every N seconds → `ScheduledExecutorService`
- Retry after delay → `ScheduledExecutorService`
- Refresh OAuth token before expiry → `ScheduledExecutorService`
- CPU-heavy data transformation → `ForkJoinPool`
