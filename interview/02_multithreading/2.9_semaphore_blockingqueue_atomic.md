## Complete Answer

### CountDownLatch:
```java
CountDownLatch latch = new CountDownLatch(3);
executor.submit(() -> { doWork(); latch.countDown(); });
latch.await(5, TimeUnit.SECONDS); // block until count = 0
```
One-time use. Use for: fan-out wait, async test synchronization, startup gates.

### Semaphore — N threads allowed simultaneously:
```java
Semaphore semaphore = new Semaphore(3); // max 3 concurrent
semaphore.acquire();
try { partnerApi.call(); } finally { semaphore.release(); }
```
vs synchronized/ReentrantLock = 1 thread at a time.
Use for: rate limiting, throttling concurrent partner calls.

### BlockingQueue — thread-safe producer/consumer:
```java
BlockingQueue<Order> queue = new ArrayBlockingQueue<>(500);
queue.put(order);  // blocks if full — natural backpressure
queue.take();      // blocks if empty
```
Implementations: LinkedBlockingQueue, ArrayBlockingQueue, PriorityBlockingQueue, SynchronousQueue.
Used internally by ThreadPoolExecutor for pending tasks.

### AtomicInteger / CAS:
```java
// CAS — single CPU instruction (CMPXCHG), no kernel involvement
if (current == expected) { current = newValue; return true; }

AtomicInteger counter = new AtomicInteger(0);
counter.incrementAndGet(); // thread-safe, no synchronized
```
- Low contention → faster than synchronized (no thread parking)
- High contention → spin loop burns CPU → synchronized wins
- Rule: single variable → Atomic*, multiple variables together → lock
