# 2.8 What is a deadlock? Describe the four conditions required and one prevention strategy.

## Your Answer
- Thread A waits for lock held by Thread B, Thread B waits for lock held by Thread A ✓

## What Was Missing / Wrong

## Complete Answer

### Definition:
A deadlock occurs when two or more threads are permanently blocked, each waiting
for a resource held by another.

### Four Coffman Conditions (ALL must hold):
1. **Mutual exclusion** — resource held by only one thread at a time
2. **Hold and wait** — thread holds a resource while waiting to acquire another
3. **No preemption** — resources cannot be forcibly taken; only voluntarily released
4. **Circular wait** — T1 waits for T2, T2 waits for T1 (the cycle)

Break any one → deadlock impossible.

### Prevention strategies:

**1. Consistent lock ordering** (breaks circular wait):
```java
void transfer(Account from, Account to) {
    Account first  = from.id < to.id ? from : to;
    Account second = from.id < to.id ? to : from;
    synchronized (first) {
        synchronized (second) { ... }
    }
}
```

**2. tryLock with timeout** (breaks hold-and-wait indefinitely):
```java
if (lock1.tryLock(100, MILLISECONDS)) {
    try {
        if (lock2.tryLock(100, MILLISECONDS)) {
            try { ... } finally { lock2.unlock(); }
        }
    } finally { lock1.unlock(); }
}
```

**3. Avoid nested locks** — acquire only one lock at a time when possible.
