# 2.7 What is the difference between `Thread.sleep()` and `Object.wait()`?

## Your Answer
- `sleep()` suspends the thread for a specific time ✓
- `wait()` parks the thread until `notify()` is called ✓
- `wait()` releases the lock ✓
- `sleep()` does not release the lock ✓
- Why `wait()` must be inside `synchronized` — not sure ✗

## What Was Missing / Wrong
- Missing: calling `wait()` outside `synchronized` throws **`IllegalMonitorStateException`**
- Missing: **spurious wakeups** — must use `while` loop, not `if`, to recheck condition

## Complete Answer

### Key differences:

| | `Thread.sleep()` | `Object.wait()` |
|---|---|---|
| Belongs to | `Thread` class | `Object` class |
| Releases lock? | **No** | **Yes** |
| Woken by | Timer expiry | `notify()` / `notifyAll()` / timeout |
| Requires synchronized? | No | **Yes** |

### Why wait() must be inside synchronized:
`wait()` releases the monitor on the object. If you don't hold the monitor,
there's nothing to release → `IllegalMonitorStateException` at runtime.

### Correct pattern:
```java
synchronized (lock) {
    while (!condition) {   // while, NOT if — spurious wakeups
        lock.wait();       // releases lock, parks thread
    }
    // do work
}
```

### Spurious wakeups:
A thread can wake up without `notify()` being called (JVM/OS-level artifact).
Always recheck the condition in a `while` loop after waking.
