## Complete Answer

### Virtual Threads (Java 21):
- Platform thread blocked on I/O = OS thread blocked, ~1MB stack wasted
- Virtual thread blocked = unmounted from carrier thread, carrier picks up other work
- Millions of virtual threads possible — carrier pool is small (= CPU cores)
- Write simple blocking code, get async-level throughput

```java
try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
    executor.submit(() -> partnerApi.getOrder(id)); // blocks but doesn't waste OS thread
}
```

### Records (Java 16):
```java
public record OrderResponse(String id, String status) {}
// Immutable, accessor methods (id(), status()), equals, hashCode, toString generated
```
Use for: DTOs, Value Objects, method parameter grouping
Don't use for: JPA entities, anything needing mutable state

### Sealed Classes (Java 17):
```java
public sealed class Shape permits Circle, Square, Triangle {}
public final class Circle extends Shape {}
// Compiler enforces exhaustive switch — no default needed
```
Use for: closed event hierarchies, partner event types

### Pattern matching instanceof (Java 16):
```java
// Before
if (shape instanceof Circle) { Circle c = (Circle) shape; }

// After
if (shape instanceof Circle c) { c.radius(); } // no cast needed

// Combined with sealed + switch (Java 21)
switch (event) {
    case OrderCreated e  -> "created: " + e.orderId();
    case PaymentFailed e -> "failed: " + e.reason();
}
```
