## Complete Answer

### Comparable — natural ordering, inside the class:
```java
public class Flight implements Comparable<Flight> {
    public int compareTo(Flight other) {
        return Integer.compare(this.price, other.price); // natural order = by price
    }
}
Collections.sort(flights); // uses natural order
```

### Comparator — external, custom ordering:
```java
Comparator<Flight> byPrice = Comparator.comparing(Flight::price);
Comparator<Flight> byDestination = Comparator.comparing(Flight::destination);
Comparator<Flight> byPriceThenDest = byPrice.thenComparing(byDestination);

flights.sort(byDestination); // overrides natural order
```

### Key distinctions:

- Location: Comparable is inside the class; Comparator is outside the class
- Number of orderings: Comparable has one (natural); Comparator can have many
- When to use: Comparable when you own the class and have one obvious order;
  Comparator when multiple orders are needed, or you don't own the class
- Interface method: Comparable uses `compareTo()`; Comparator uses `compare()`

### Comparator wins over Comparable:
```java
// Flight implements Comparable (by price)
Collections.sort(flights, Comparator.comparing(Flight::destination));
// sorts by destination, ignores compareTo()
```
