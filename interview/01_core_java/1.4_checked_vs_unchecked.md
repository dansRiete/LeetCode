# 5. What is the difference between checked and unchecked exceptions? When do you use each?

## Your Answer
- Checked exceptions are forced to be handled at compile level ✓
- Checked = contract / expected cases ✓
- Unchecked = rare edge cases where nothing can be done ✓

## What Was Missing / Wrong
- Solid answer overall
- Could be more precise: unchecked = **programming errors** or
  **unrecoverable situations**, not just "rare edge cases"
- Missing: modern frameworks prefer unchecked exceptions — worth mentioning
- Missing: checked exceptions inside lambdas must be caught or wrapped

## Complete Answer

### Checked — caller can reasonably recover:
```java
// forces caller to handle — it's part of the contract
public Flight fetchFlight(String id) throws IOException, FlightNotFoundException {
    ...
}

// caller must handle
try {
    Flight f = fetchFlight("123");
} catch (IOException e) {
    // retry
} catch (FlightNotFoundException e) {
    // return 404
}
```

### Unchecked (RuntimeException) — use for:
- **Programming errors**: `NullPointerException`, `IllegalArgumentException`,
  `IndexOutOfBoundsException`
- **Unrecoverable situations**: where catching and continuing leaves system in
  inconsistent state

```java
public void setPrice(int price) {
    if (price < 0) throw new IllegalArgumentException("Price cannot be negative: " + price);
}
```

### Modern preference — unchecked:
- Spring, most libraries throw unchecked exceptions
- Checked exceptions pollute method signatures
- Forces callers to handle things they can't do anything about
- Impractical inside stream pipelines:
```java
// DOES NOT COMPILE — checked exception in lambda
records.stream().map(r -> parseRecord(r)); // if parseRecord throws IOException

// workaround — wrap in unchecked
records.stream().map(r -> {
    try { return parseRecord(r); }
    catch (IOException e) { throw new RuntimeException(e); }
});
```

### Rule of thumb:

- Caller can recover → use Checked
- Programming mistake → use Unchecked (RuntimeException)
- Framework/library code → use Unchecked
- Inside lambdas/streams → use Unchecked
