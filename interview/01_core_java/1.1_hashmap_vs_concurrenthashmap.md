# 1. What is the difference between HashMap and ConcurrentHashMap?

## Your Answer
- ConcurrentHashMap is synchronized and safe to use in multithreaded environment ✓
- Uses CAS mechanism under the hood ✓

## What Was Missing / Wrong
- CAS is only part of the story — also uses **synchronized on individual buckets**
  for complex operations
- Key distinction missed: `Hashtable` and `Collections.synchronizedMap()` lock the
  **entire map**, ConcurrentHashMap only locks the **affected bucket** —
  much better throughput
- **Compound operations are not atomic** — `get()` + `put()` together are not thread
  safe even though each is individually safe
- Use `putIfAbsent()`, `computeIfAbsent()`, `compute()`, `merge()` for atomic
  compound operations

## Complete Answer

**HashMap vs ConcurrentHashMap:**

- Thread safe: No vs Yes
- Locking: None vs Bucket-level
- Null keys: Yes vs No
- Performance: Faster (single thread) vs Slower but concurrent

### How ConcurrentHashMap achieves thread safety:
- **CAS** — for inserting into an empty bucket
- **synchronized on bucket** — for chaining within a bucket
- Result: multiple threads can read/write **different buckets simultaneously**

### The compound operation trap:
```java
// NOT thread safe even with ConcurrentHashMap
String val = map.get("key");
if (val == null) {
    map.put("key", "value"); // race condition here
}

// thread safe — atomic
map.computeIfAbsent("key", k -> "value");
map.putIfAbsent("key", "value");
```
