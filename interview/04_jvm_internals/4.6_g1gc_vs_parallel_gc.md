# 4.6 — G1GC vs Parallel GC

## Question
What is G1GC and why did it become the default in Java 9? What problem does it solve vs Parallel GC?

## Complete Answer

### Parallel GC:
- Fixed Young/Old heap regions
- Maximizes throughput — long, unpredictable STW pauses
- Pauses can be 5-10s on large heaps — unacceptable for latency-sensitive services

### G1GC (Garbage First):
- Heap split into many small equal-sized regions (~2MB), dynamically assigned as Eden/Survivor/Old
- Collects regions with most garbage first
- Targets configurable pause time: `-XX:MaxGCPauseMillis=200`
- Trades some throughput for predictable, bounded pauses

### Why default since Java 9:
Most services care more about consistent latency than raw throughput.
200ms predictable pause >> random 5s pause.

### GC evolution:
- Parallel GC → max throughput, unpredictable pauses
- G1GC → balanced, predictable pauses (Java 9+ default)
- ZGC / Shenandoah → sub-millisecond pauses (Java 15+, extreme latency requirements)
