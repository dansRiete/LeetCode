# 4.4 — Soft, Weak, and Phantom References

## Question
What is the difference between soft, weak, and phantom references? When would you use WeakReference?

## Complete Answer

### Reference strength (strongest → weakest):
| Type | Collected when | Use case |
|------|---------------|----------|
| Strong | Never (normal reference) | Everything by default |
| Soft | Only when JVM is low on memory | Memory-sensitive caches |
| Weak | Next GC cycle, regardless of memory | Canonical maps, listener registries |
| Phantom | After finalization | Cleanup/resource tracking |

### WeakReference example:
```java
WeakReference<MyObject> ref = new WeakReference<>(obj);
obj = null; // strong reference dropped
// next GC → ref.get() returns null
```

### WeakHashMap — auto-removes entries when key no longer strongly referenced:
```java
WeakHashMap<Key, Value> map = new WeakHashMap<>();
// entry disappears when no strong reference to key exists
```

### Why WeakHashMap is dangerous as a general cache:
- Keys collected at any GC cycle if no other strong reference exists
- Entries disappear unpredictably — cache misses at random
- No control over eviction timing, size, or TTL

### Use instead:
- **Caffeine** or **Guava Cache** — explicit max size, TTL, eviction policies
