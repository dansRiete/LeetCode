## Complete Answer

### Mock vs Stub vs Spy:
- **Stub** — returns canned data, no verification: `when(x).thenReturn(y)`
- **Mock** — verifies interactions: `verify(x, times(1)).method()`
- **Spy** — wraps real object, selectively override: `Mockito.spy(realObject)`

### Test Pyramid:
```
    E2E (~10%)       — slow, expensive, few
  Integration (~20%) — medium speed, some
Unit Tests (~70%)    — fast, cheap, most
```
Anti-pattern: Ice Cream Cone — heavy E2E, weak unit coverage.

### Async testing:
```java
// CompletableFuture.get() — when you own the future
future.get(5, TimeUnit.SECONDS);

// CountDownLatch — when you don't own the future
CountDownLatch latch = new CountDownLatch(1);
doAnswer(inv -> { latch.countDown(); return null; }).when(repo).save(any());
assertTrue(latch.await(5, SECONDS));

// Awaitility — most readable
await().atMost(5, SECONDS).untilAsserted(() -> verify(repo).save(any()));
```

### Flaky tests (non-deterministic — passes sometimes, fails sometimes):
| Cause | Fix |
|-------|-----|
| Timing/async | Awaitility / CountDownLatch instead of sleep |
| Shared static state | Reset in @BeforeEach/@AfterEach |
| Test order dependency | Make each test fully independent |
| External dependency | Mock external calls |
| LocalDate.now() | Inject fixed Clock |

```java
// Inject Clock for deterministic date tests
Clock fixed = Clock.fixed(Instant.parse("2026-01-01T00:00:00Z"), ZoneOffset.UTC);
assertTrue(booking.isExpired(fixed));
```
